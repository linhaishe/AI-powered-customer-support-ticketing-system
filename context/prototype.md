# 天猫海外 AI 客服服务中台原型文档

## 1. 原型目标

第一阶段原型目标是展示一个真实可信的天猫海外 AI 客服工作台。用户打开首页后，不需要进入登录、介绍页或配置流程，可以直接看到客服处理跨境工单的核心界面。

原型重点展示：

- 工单队列。
- 跨境业务上下文。
- AI 辅助处理面板。
- 人工确认和风险提示。

## 2. 页面清单

第一阶段只做一个页面：

- 首页：客服工作台。

路径：

```txt
/
```

页面标题：

```txt
天猫海外 AI 客服服务中台
```

## 3. 首页整体布局

页面采用后台工作台布局。

```txt
┌──────────────────────────────────────────────────────────────────────┐
│ 顶部栏：天猫海外 AI 客服服务中台 / 日期 / 当前角色                  │
├──────────────────────────────────────────────────────────────────────┤
│ 指标区：今日工单 | AI采纳率 | 高风险工单 | 平均处理时长             │
├───────────────┬──────────────────────────────────────┬───────────────┤
│ 左侧工单队列  │ 中间工单详情                         │ 右侧AI辅助面板 │
│               │                                      │               │
│ 工单A         │ 消费者问题                           │ AI摘要         │
│ 工单B         │ 历史沟通                             │ 风险等级       │
│ 工单C         │ 订单卡片                             │ 处理建议       │
│ 工单D         │ 物流轨迹                             │ 知识引用       │
│               │ 售后状态                             │ 回复草稿       │
│               │ 商家信息                             │ 操作按钮       │
└───────────────┴──────────────────────────────────────┴───────────────┘
```

## 4. 顶部栏

### 内容

- 产品名称：天猫海外 AI 客服服务中台。
- 副标题：跨境服务工单智能辅助工作台。
- 当前角色：平台客服。
- 当前日期或工作班次。

### 设计目的

顶部栏要让用户立刻理解这是一个客服业务系统，而不是通用 AI 聊天页面。

## 5. 指标区

### 指标卡片

四个指标卡片横向排列：

1. 今日服务工单：`128`
2. AI 建议采纳率：`73%`
3. 高风险工单：`12`
4. 平均处理时长：`8.6 分钟`

### 交互

第一阶段不需要点击跳转，指标只用于展示工作台氛围。

### 视觉要求

- 高风险工单指标需要更醒目。
- 指标文案简洁。
- 不使用过大的营销型标题。

## 6. 左侧工单队列

### 信息结构

每条工单展示：

- 消费者名称或脱敏标识。
- 工单标题。
- 问题类型。
- 订单号。
- 状态。
- 风险等级。
- 更新时间。

### 示例

```txt
Maria G.
包裹清关停滞，消费者要求解释
物流延迟 / 高风险
ORDER-889271 / 12分钟前
```

```txt
Kenji T.
退款申请已提交但未到账
退款进度 / 中风险
ORDER-773820 / 26分钟前
```

```txt
Aisha R.
咨询进口税费是否由平台承担
清关税费 / 中风险
ORDER-910244 / 45分钟前
```

### 交互

- 默认选中第一条工单。
- 点击工单后，整行高亮。
- 中间详情和右侧 AI 面板同步切换。

## 7. 中间工单详情

中间区域是客服做判断的主区域。

### 7.1 消费者问题卡片

展示：

- 消费者原始描述。
- 消费者语言。
- 消费者国家或地区。
- 情绪提示。

示例文案：

```txt
我的包裹已经超过预计送达时间 7 天，物流一直停在清关状态。
我已经联系过两次客服，但没有得到明确答复。请告诉我什么时候能收到，
如果不能送达，我希望获得退款或补偿。
```

### 7.2 历史沟通卡片

展示最近 2 到 3 条沟通记录。

内容包括：

- 消费者追加说明。
- 客服历史回复。
- 系统事件。

### 7.3 订单信息卡片

字段：

- 订单号。
- 商品名称。
- 支付状态。
- 发货状态。
- 订单金额。
- 下单时间。
- 收货国家或地区。

### 7.4 物流轨迹卡片

字段：

- 物流单号。
- 承运商。
- 当前节点。
- 最近更新时间。
- 预计送达时间。
- 清关状态。
- 异常原因。

轨迹展示示例：

```txt
已揽收 -> 国际运输中 -> 到达目的国 -> 清关处理中 -> 轨迹 5 天未更新
```

### 7.5 售后状态卡片

字段：

- 售后单号。
- 售后类型。
- 当前状态。
- 商家处理结果。
- 平台介入状态。
- 处理截止时间。

如果没有售后单，显示：

```txt
当前订单暂无售后申请。
```

### 7.6 商家信息卡片

字段：

- 店铺名称。
- 服务评分。
- 平均响应时长。
- 近期投诉量。
- 高频问题标签。

## 8. 右侧 AI 辅助面板

右侧是核心展示区域。

### 8.1 AI 摘要

展示一段 2 到 3 行摘要。

示例：

```txt
消费者反馈订单超过预计送达时间，物流已到达目的国但清关状态 5 天未更新。
消费者情绪不满，已多次咨询，希望获得明确送达时间或退款补偿方案。
```

### 8.2 分类和情绪

展示：

- 问题类型：物流延迟 / 清关异常。
- 情绪：不满。
- 风险等级：高。

### 8.3 处理建议

展示 3 到 5 条建议。

示例：

- 先解释当前物流节点和清关可能原因。
- 避免承诺具体送达日期。
- 告知消费者已升级物流专员跟进。
- 若 48 小时内仍无更新，建议转主管复核补偿策略。
- 回复中引用跨境物流清关说明。

### 8.4 知识引用

展示 AI 使用的依据。

示例：

```txt
《跨境物流清关时效说明》
适用地区：欧盟
说明：目的国清关通常需要 3 到 7 个工作日，节假日或海关抽检可能延长。
```

```txt
《物流异常客服处理 SOP》
说明：轨迹超过 5 天未更新时，客服应升级物流专员并向消费者说明后续跟进节点。
```

### 8.5 多语言回复草稿

展示可编辑回复草稿。

示例：

```txt
Hi Maria, we are sorry for the inconvenience caused by the delivery delay.
Your package has arrived in the destination country and is currently waiting
for customs clearance. Customs processing time may vary depending on local
inspection and holiday schedules.

We have escalated this case to our logistics specialist for further tracking.
If there is still no update within the next 48 hours, we will review the case
again and advise you on the next available options. Thank you for your patience.
```

### 8.6 安全提示

展示：

```txt
AI 提醒：当前场景涉及可能的赔付或退款诉求，请勿在未复核前承诺补偿金额或具体退款结果。
```

### 8.7 操作按钮

按钮：

- 采纳草稿。
- 编辑草稿。
- 忽略建议。

交互：

- 点击采纳后显示“已采纳，等待发送前确认”。
- 点击编辑后草稿进入可编辑状态。
- 点击忽略后显示“已忽略，客服将按人工判断处理”。

## 9. 状态和标签规范

### 工单状态

- 新建。
- 处理中。
- 等待消费者。
- 已升级。
- 已解决。

### 优先级

- 低。
- 中。
- 高。
- 紧急。

### 风险等级

- 低风险。
- 中风险。
- 高风险。

高风险应使用更醒目的样式，但不使用干扰阅读的大面积警告色。

## 10. 响应式要求

第一阶段以桌面端为主。

推荐最小宽度：

```txt
1280px
```

桌面布局：

- 左侧队列固定宽度。
- 中间详情自适应。
- 右侧 AI 面板固定宽度。

窄屏处理：

- 指标区换行。
- 三栏可改为纵向堆叠。
- AI 面板保持在详情之后。

## 11. 原型验收标准

- 首页直接进入客服工作台。
- 用户能看到平台客服业务语境。
- 用户能切换 3 到 4 条工单。
- 每条工单有独立上下文和 AI 输出。
- 主场景清楚体现跨境物流、清关、投诉、风险和多语言回复。
- AI 辅助面板明确体现“辅助客服，不直接替代人工决策”。
- 页面没有明显信息重叠、文字溢出或默认模板残留。

## 12. 演示讲述顺序

面试或作品集演示时建议按这个顺序讲：

1. 这是一个跨境电商客服 AI 服务中台。
2. 左侧是平台客服待处理工单。
3. 中间聚合订单、物流、售后、商家和规则上下文。
4. 右侧 AI 自动总结问题、识别风险并生成处理建议。
5. AI 回复草稿必须由客服采纳或编辑后发送。
6. 高风险动作不由 AI 直接执行，系统保留人工确认和审计空间。
7. 当前版本使用 mock 数据，后续可替换为 BFF、数据库、AI SDK 和知识检索。
